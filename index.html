Definitions

HTML: Hypertext Markup Language is the coding language used to describe the content of websites

CSS: Cascading Style Sheets (CSS) is a stylesheet language used to describe the presentation of a document written in HTML or XML (including XML dialects such as SVG, MathML or XHTML). CSS describes how elements should be rendered on screen, on paper, in speech, or on other media.

Javascript: JavaScript is a dynamic programming language that's used for web development, in web applications, for game development, and lots more. It allows you to implement dynamic features on web pages that cannot be done with only HTML and CSS

Syntax: When writing in HTML, the tags used to define the purpose of the text follow a specific syntax. Elements have a start tag to indicate where they begin. Non-void elements have an end tag to indicate where they end. tag names are used within element start tags and end tags to give the element's name.

Accessibility: Web accessibility means everyone, including users with disabilities, will have an ideal experience and can access the information equally.

WAI-ARIA:Web Accessibility Initiative WAI) has a specification called Accessible Rich Internet Application (ARIA)

Commit messages best practises

  • Minimum 3 words & 10 characters
  • Proper spelling & grammar
  • Capital sarting letter, no period
  • Must start with an imperative verb: Create, Fix, Add, Solve, etc.
  • Pretend every commit starts with the phrase: “This commit will…”

Link Text

The opening <a> tells where the link should start and the closing </a> indicates where the link ends. To create a hyperlink, you should use the <a> tag and href attribute, the value of which is the URL, or location, where the link is pointing to.

<a href="link text" =""description"</a>

Link Image

Images in websites are not embedded in the website but linked to by the HTML document. Images need two attributes to specify extra details about the image: the location of the image and the alternative content.

<img src="image link" alt="image description">

Footers:

Many webpages end their pages with a footer at the bottom. This often includes lists of links, copyright information, extra information about the company, terms and services.

</footer> close tag

Main:

The main element wraps around the main content of the page. It’s only used once per page and it tells the browser that this is where the main content is.

<main> open tag
</main> close tag

Section:

Section is used to wrap around related pieces of content. Usually each section has its own heading.

<section> open tag
</section> close tag

Article

The article element represents a self- contained composition in a web page, which is intended to be independently distributable or reusable. Examples include blog post, product card, interactive widget, etc...

<article> open tag
</article> close tag

Nav:

The nav element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents.

<nav> open tag
</nav > close tag

Aside:

The aside element represents a portion of a document whose content is only indirectly related to the document's main content. This content will frequently be presented as a sidebar on a website.

<aside > open tag
</aside> close tag

Accessibility

Websites must comply with the Web Content Accessibility Guidelines (WCAG) 2.0

<header role = "banner"> open tag

This role is for defining an element as the masthead, added to the primary <header>

<nav role = "navigation"> open tag

This role is for defining an element as the primary navigation, added the the top <navigation>

<main role = "main"> open tag

This role is for defining an element as being the main content of the page, added to the <main>

aside role = "complimentary"> open tag

This role is for defining an element as being content related to the primary content, usually added to an <aside>

<footer role = "contentinfo"> open tag

This role is for defining an element as information about the content: e.g. copyright, terms, privacy policy, usually added to a <footer>
